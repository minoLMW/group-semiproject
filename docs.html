HTML5의 문법을 기본으로 따른다.
인코딩
대소문자를 구별하여 CSS와 동일한 인코딩을 지정한다.
문자
태그명, Attribute명과 값은 소문자와 숫자, 대시(-)만 사용하되 alt값은 자유롭게 사용한다.
따옴표
큰따옴표만 사용한다.
개행
코드 한줄이 너무 길어지지 않도록 개행을 해야한다.
태그 내부에 문자만 있는 경우 한줄로 표기하고 태그와 함께 있는 경우에는 개행한다.
개행은 코드의 가독성을 높이기 위해 가로방향으로 너무 길어지지 않게 하기 위한 수단으로 가급적 지키되 충분한 의도가 있는 경우라면 규칙에서 벗어나도 무방하다.
br 의 경우 반드시 개행하고 문단의 흐름안에 있는 인라인 태그의 경우 개행할 필요는 없다.
들여쓰기
한번 들여쓰기는 공백4칸으로 정의하고 중첩이 깊어질때마다 한번씩 들여쓴다.
주석
주석은 코드 끝부분에 작성하지 않으며 해당하는 라인의 직전라인에 작성한다.
한페이지에 여러 케이스가 함께 있는 경우 표시한다.
상시 운영시에 신규를 제외한 기존 페이지에서 수정은 다음과 같이 표시한다.
같은 날짜에 변경된 다른 수정사항들을 명확히 구분 할 수 있고 같은 곳이 수정 되더라도 어느 요건에서 수정사항이 발생했는지 알 수 있다.
CSS
문자
  • 영문은 소문자로 작성한다.
  • 가상 요소(Pseufo Element)와 Content 값은 자유롭게 지정한다.
  • font-family의 값도 폰트이름에 따라 자유롭게 지정한다.
따옴표
이름에 공백이 포함된 폰트나 한글 이름의 폰트는 큰따옴표를 사용한다.
background의 url값의 따옴표는 생략한다.
세미콜론
모든 Property의 마지막에는 세미콜론을 삽입한다.
들여쓰기
  • 한번 들여쓰기는 탭 4칸으로 정의한다.
  • :root와 @media, @keyframes 설정외에는 들여쓰기 하지 않는다.
공백
선택자와 중괄호 사이에 공백을 한칸 삽입한다.
자식선택자, 일반형제선택자, 인접형제선택자의 경우 이전과 다음에 공백을 각각 한 칸씩 삽입한다.
중괄호 앞과 뒤, 콜론( : )의 이전과 다음은 공백없이 붙인다.
세미콜론( ; )과 다음 property 사이에 공백을 한칸 삽입한다.
개행
:root와 @media인 경우에 중괄호가 시작할때 한번, 중괄호가 끝나기전에 한번 개행한다.
서로 다른 선택자의 경우 개행으로 구분한다.
빈줄
@media가 연속으로 올 경우 빈줄 삽입하여 구분한다.
Property 선언 순서
Property 선언 순서
1. 레이아웃 content, display, flex, visibility, opacity, overflow, float, clear, position, top, right, bottom, left, z-index
2. 박스모델 width, min-width, max-width, height, min-height, max-height, margin, padding, border, border-radius, outline, background, box-shadow
3. 글꼴 color, font-style, font-variant, font-weight, font-size, line-height, font-family, letter-spacing, text-transform, text-align, vertical-align, text-shadow
4. 기타 그외 모든 속성
5. 변형 & 모션 transform, transition, animate
관련 속성 그룹핑
몇몇 속성은 낱개로 사용할 때는 기타의 위치에 선언하지만 특정한 용도가 있을 경우 관련된 속성끼리 그룹핑한다.
말줄임과 관련된 경우 white-space는 overflow속성 다음에 위치한다.
말줄임과 관련이 없는 경우 white-space는 글꼴 속성보다 다음인 기타위치에 선언한다.
방향에 따른 순서
방향과 관련 된 스타일인 경우 시계방향(top, right, bottom, left)의 순서로 선언한다.
약식속성
약식 속성을 사용시 지정하지 않은 값을 의도치 않게 초기화하는 경우가 발생할 수 있으므로 사용에 주의가 필요하다.
border는 width, style, color의 순서로 선언한다.
background는 개별적으로 선언하고 두개이상의 속성을 선언할 경우에만 약식을 사용한다.
font의 경우 대상 태그가 최하위 요소일 경우에만 제한적으로 사용을 권장한다.
축약
16진수 컬러 값은 세자리로 축약한다.
값이 0인 경우 단위는 생략한다.
주석
구분이 필요한 상황에만 제한적으로 사용하고 가급적 사용하지 않는다.
단위
CSS의 수치 단위는 모바일 맞춰서 rem , em , vw , vh 자유롭게 사용하되 px 은 1px 수치를 제외하고 사용하지 않는다.

클래스 작명 규칙
옵션 클래스
  • 컴포넌트 클래스와 옵션 클래스를 구분없이 사용할 경우 급속히 클래스가 고갈되는 문제가 있고 비슷한 이름의 클래스가 양산되는 문제가 있다.
  • 상태나 성격을 바꾸기 위해 사용되는 클래스에는 prefix로 대시(-)를 붙인다.
두 단어간의 연결
단어를 연결할 때는 대시(-)를 사용한다.
약어 사용을 지양
  • 클래스를 과도하게 줄이는 경우 작업자마다의 해석이 달라질 수 있어 의미 해석의 오류가 발생할 수 있으므로 약어의 사용을 최대한 지양한다.
  • 서로 다른 축약 방식으로 비슷한 이름이 양산될 수 있어 코드 해석에 방해가 될 수 있다.
  • 모든 단어는 온전한 단어로 표기를 하는 것을 지향하되 너무 길거나 풀어쓰는 것보다 약어 자체가 더 많이 사용되는 경우는 약어를 사용해도 좋다.
예약어
레이아웃이나 공통UI에 사용되었거나 전역에서 자주 사용될 수 있는 이름을 예약어로 지정하여 사용을 금지하거나 제한적으로 사용하도록 규칙을 정한다.
사용 금지
  • 태그명과 동일한 클래스명은 공툥UI외에는 사용할 수 없습니다.
  • 레이아웃에 사용된 이름은 사용할 수 없습니다.
